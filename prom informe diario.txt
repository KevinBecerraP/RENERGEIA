Contexto funcional
RenergeIA es una plataforma interna para control de proyectos EPC fotovoltaicos.
El objetivo es centralizar el control operativo, documental y gerencial del proyecto, reemplazando procesos dispersos en Excel, WhatsApp, correos y archivos independientes.
La empresa actualmente utiliza una plantilla de Informe Diario en Excel, pero NO queremos replicar el Excel como una página web.
La plantilla del Informe Diario se entrega únicamente como referencia funcional para entender:
•	Qué información se captura.
•	Qué indicadores se calculan.
•	Qué gráficas se generan.
•	Qué reportes necesita la empresa.
•	Cómo se relacionan avances, cronograma, curva S y reportes.
________________________________________
Principio principal
El cronograma / WBS es la fuente maestra.
El Informe Diario no debe permitir crear actividades nuevas.
Las actividades del Informe Diario deben provenir exclusivamente del módulo WBS / Actividades.
Si una actividad está activa en el cronograma, debe aparecer en el Informe Diario.
Si una actividad está desactivada o marcada como No Aplica, no debe aparecer en el Informe Diario.
Si se activa nuevamente, debe volver a aparecer.

Objetivo del módulo Informe Diario
Construir un módulo que permita registrar diariamente el avance real de las actividades activas del cronograma.
Para cada fecha, el sistema debe permitir crear un Informe Diario asociado a un proyecto.
Dentro de ese informe deben cargarse automáticamente las actividades activas según el cronograma.
Para cada actividad se debe poder registrar:
•	Cantidad ejecutada del día.
•	Porcentaje de avance reportado.
•	Observaciones.
•	Restricciones.
•	Personal asociado.
•	Equipos utilizados.
•	Fotografías.
•	Novedades.
•	Horas afectadas por lluvia o clima.
•	Responsable que reporta.
________________________________________
Cálculo de avance esperado
El sistema debe calcular automáticamente cuánto debería llevar una actividad según sus fechas planificadas.
Ejemplo:
Actividad: Hincado
Fecha inicio planificada: 01/06/2026
Fecha fin planificada: 30/06/2026
Fecha del informe: 15/06/2026
El sistema debe calcular:
Avance esperado: 50%
Si el usuario reporta:
Avance real: 35%
El sistema debe mostrar:
Avance esperado: 50%
Avance real: 35%
Desviación: -15%
Estado: Atrasado
Alerta: Actividad con retraso frente al cronograma.

Flujo de validación
El informe diario debe tener estados:
•	Borrador.
•	En revisión.
•	Aprobado.
•	Rechazado.
•	Anulado.
Reglas:
•	El residente o responsable registra el informe.
•	El director de proyecto revisa.
•	Si aprueba, el informe se bloquea.
•	Si rechaza, vuelve con observaciones.
•	Toda modificación debe quedar con trazabilidad.
•	Un informe aprobado no se edita directamente; debe generar nueva versión o corrección.
________________________________________
Alertas automáticas
El sistema debe generar alertas cuando:
•	Avance real sea menor al avance programado en más de 5 puntos porcentuales.
•	Una actividad activa no tenga reporte diario.
•	Una actividad crítica esté atrasada.
•	Una restricción crítica esté vencida.
•	La productividad del día sea inferior a la esperada.
•	Exista afectación por lluvia.
•	El avance acumulado supere el 100%.
•	Se registre una inconsistencia mayor al 30% respecto al día anterior.
